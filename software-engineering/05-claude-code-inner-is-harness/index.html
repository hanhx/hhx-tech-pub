第一维度：信任门控

先确认身份，再给工具

源码注释里反复出现一个词：deferred。所有能力扩展，全都在信任验证通过之后才启动。

deferred
MCP 资源预取
deferred until after trust dialog
deferred
插件加载
deferred from setup()
deferred
LSP 初始化
intentionally deferred
before trust
信任前只做三件事
环境检测  ·  CLI 解析  ·  权限校验——不加载任何可以执行动作的子系统
关键洞察：Harness 不是 AI 跑起来之后才开始的事。
启动流程本身就是第一道 Harness。

第二维度：工具约束

最危险的工具，约束最重

第一层 · 全局过滤

ToolPermissionContext

用 allow / deny / ask 三套规则过滤每一次工具调用——所有工具都要过这道闸

第二层 · 单工具加固

BashTool 九个专属校验

BashTool 是最强大也最危险的工具，在全局过滤之上还有九个独立校验模块：

bashSecurity bashPermissions pathValidation readOnlyValidation modeValidation destructiveCommandWarning shouldUseSandbox sedValidation commandSemantics

其中 bashSecurity 单文件就定义了 23 种安全检查

第三层 · 极端裁剪

--bare 模式

开启后整个工具池只保留三个工具：BashTool · FileReadTool · FileEditTool，同时跳过插件、Hook、LSP 等所有非核心子系统

工程直觉：最大的价值不是让 AI 能做所有事，而是用约束让它只在安全边界内行动。

第三维度：运行边界

熔断、压缩、持久化——三道防线

1

硬熔断

maxTurns 和 taskBudget 两道硬约束，触发即停止，记录停止原因：
end_turn / max_turns / token_budget

2

软压缩

autoCompact 系统在接近上下文窗口上限时自动压缩对话历史，不停止运行，让长任务能继续推进

3

断点恢复

每次 session 完整序列化到磁盘：session_id · 全部消息记录 · token 统计。熔断后下一轮从中断点接上

与 04 期同一工程直觉：跟 Initializer Agent 留下 progress.txt 和 git commit 基线是一回事——让下一轮有据可查。

第四维度：分工 Agent

不是一个模型在跑，而是一支队伍

AgentTool 里有六个内置 Agent，大部分有 feature flag 门控，可动态开关：

generalPurpose
通用 Agent
处理通用任务的基础执行单元
statuslineSetup
状态栏配置
负责状态信息的展示与配置
explore
代码探索
专职理解和导航代码库结构
plan
方案规划
拆解任务、生成执行计划
claudeCodeGuide
使用引导
用户上手和使用场景引导
verification
实现验证
专职验收，确保实现符合预期

🔀 forkSubagent

子 Agent 可以在运行时动态分裂——不是静态流水线，而是可伸缩的工作单元

♻️ resumeAgent

子 Agent 可从磁盘恢复中断状态继续执行——断点不丢，队伍可接力
