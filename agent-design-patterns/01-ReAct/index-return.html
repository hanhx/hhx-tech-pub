正常完成

Happy Path
✅ completed

模型返回的内容中没有 tool_use 块（needsFollowUp === false），说明任务做完了，不需要再调工具。这是 ReAct 循环唯一的正常出口。

if (!needsFollowUp) return { reason: 'completed' } // line 1264 & 1357
两处触发：流式结束后 (L1264)、stop hook 放行后 (L1357)

安全阀 — 防止无限循环

Guard Rails
🔄 max_turns

调用方通过 maxTurns 参数设定循环上限。每轮 turnCount + 1，超过上限就强制退出。这是"如果永远不 completed 怎么办"的直接答案。

const nextTurnCount = turnCount + 1 if (maxTurns && nextTurnCount > maxTurns) { return { reason: 'max_turns', turnCount: nextTurnCount } // line 1711 }
SDK / headless 模式的核心防护
📏 blocking_limit

每轮开始前检查上下文 token 数。当消息历史撑满上下文窗口（且 auto-compact 未开启），直接拒绝继续。

const { isAtBlockingLimit } = calculateTokenWarningState(...) if (isAtBlockingLimit) { return { reason: 'blocking_limit' } // line 646 }
上下文窗口的硬刹车
🛑 stop_hook_prevented

stop hook 机制可以在模型"以为做完了"时阻止退出并注入额外指令。但如果 stop hook 本身判定不应继续，也会终止循环。

return { reason: 'stop_hook_prevented' } // line 1279
stop hook 的安全退出
🪝 hook_stopped

工具执行完毕后，canUseTool hook 判定不应继续（如权限拒绝、安全策略触发）。与 stop_hook_prevented 不同——后者在"无工具调用"路径，这个在"工具执行后"路径。

if (shouldPreventContinuation) { return { reason: 'hook_stopped' } // line 1520 }
工具执行后的 hook 拦截

用户中断

Abort
⏹️ aborted_streaming

用户在模型流式输出阶段按下 Ctrl+C 或触发 abort signal。流式立即终止，为未完成的 tool_use 块补全 error tool_result。

if (toolUseContext.abortController.signal.aborted) { yield* yieldMissingToolResultBlocks(...) // 补全孤立的 tool_use return { reason: 'aborted_streaming' } // line 1051 }
callModel 之后、runTools 之前检查
🔧 aborted_tools

用户在工具执行阶段中断。已经在跑的工具会收到 abort signal，尚未开始的工具跳过执行。

if (toolUseContext.abortController.signal.aborted) { return { reason: 'aborted_tools' } // line 1515 }
runTools 之后检查

错误退出

Error
💥 model_error

API 调用失败、网络超时、服务端错误等。catch 块捕获后直接退出，不会无限重试。

catch (error) { return { reason: 'model_error', error } // line 996 }
API 层的最终兜底
📄 prompt_too_long

服务端返回 prompt 过长错误，且 reactive compact / context collapse 恢复失败后退出。

return { reason: 'prompt_too_long' } // line 1175 & 1182
reactive compact 救不回来时的退路
🖼️ image_error

图片尺寸超限或 resize 失败时退出。

return { reason: 'image_error' } // line 977
ImageSizeError / ImageResizeError

为什么幻灯片里只写了 completed？

因为 completed 是 ReAct 模式本身的退出条件——模型认为任务完成、不再调用工具。
其余 7 种退出都是工程层的安全阀：防无限循环（max_turns）、防上下文溢出（blocking_limit）、hook 拦截（stop_hook_prevented / hook_stopped）、响应用户中断（aborted_*）、处理不可恢复错误（model_error）。

while(true) 从来不是真正的死循环——生产代码里，至少有 8 道出口保证它一定会停。
