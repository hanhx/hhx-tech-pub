🎯 顶层结论
Claude Code 的关键，不是先给代码库建一套固定索引，而是把上下文工程拆成三层： Memory Files 先立规矩， 工具闭环 负责现场取证， Context Engineering 负责把长链路继续跑下去。

① 先立规矩：Memory Files

Claude 不是一上来把仓库全塞进脑子，而是先把项目规矩和记忆入口装进每次会话的起手上下文。
起手就带上
CLAUDE.md：你手写的项目规则，比如结构、编码规范、架构约束。MEMORY.md 前 200 行：Claude 自动维护的经验索引，告诉模型“这是什么项目、之前踩过哪些坑、细节该去哪里找”。
用到再去读
Topic files（如 debugging.md 调试经验）：更细内容按主题拆开，不在启动时全量加载，等当前任务真用到相关主题时再按需读取。

② 真正干活：Main Agent 调度

Main Agent / 主代理
结合 CLAUDE.md / MEMORY.md 理解任务目标、项目边界和语境，决定下一步走哪条路。
↓↓
Tools / 工具取证
读文件、改文件、跑命令、看报错、跑测试，把"猜"变成"查证"。
Sub-agents 按需
局部深挖或并行探索，在独立上下文窗口做聚焦任务，返回浓缩摘要。
↘↙
Findings / 关键发现
工具结果 + 子代理摘要压成高信息密度证据，回流给主代理。
↓
Patch Plan / 补丁方案
基于证据收敛到相关代码区域，形成最小改动方案。
→
Verification / 验证回环
跑测试 / 命令验证改动是否成立。

③ 长会话工程化：大窗口 + 三套管理机制

200K+ 只是容量上限，真正让长会话跑稳的，是缓存、编辑、压缩这三套机制。
Prompt Caching
重复前缀复用：同一大段稳定上下文不必每轮重算，适合长多轮对话和大量背景信息。
Context Editing
选择性保留真正重要的内容，把无关工具结果、旧噪声、无效历史剥出去。
Compaction
会话快顶到窗口时，把旧内容压缩成摘要，再让新会话继续往前跑。
200K+ = 容量上限 Caching = 重复前缀复用 Editing = 清噪声 Compaction = 压旧历史
