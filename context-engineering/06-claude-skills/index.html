从每项目手工造轮子 → 标准化可复用引擎

每项目手动写 200 行索引，维护成本高，tool call 路径硬编码，重复劳动
Skills 标准化解法
YAML frontmatter 声明即用，自动扫描匹配，执行后零污染回收，开箱即用
演进价值
1 个 Skills 全项目复用，省时 80%，生态共享取代手工重复造轮子

③ 动态化带来的工程收益

从静态到动态，Main Agent 调度不变，但工程价值大幅提升
零污染上下文
100t（frontmatter）→ 5kt（正文）→ 200t（摘要），用完即收，上下文始终干净
零部署零配置
YAML 即声明，.md 即执行说明。放进 skills/ 目录就能用，不像 MCP 需部署服务
静态 + 动态互补
Memory Files 做目录（"有什么"），Skills 做执行（"怎么用"），缺一不可
100t → 5kt → 200t 零污染回收 静态 + 动态互补
💡 Skills vs Memory Files vs MCP
Memory Files 静态规则目录：告诉 Claude"项目里有什么"。启动时加载，不执行。
Skills 动态执行模块：教 Claude"怎么用技能"。按需加载，零部署，零污染。
MCP 外部接口标准：只是通路，部署复杂。2026 趋势：MCP 做基础设施，Skills 做标准接口。
